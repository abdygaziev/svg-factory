# White Board History Asset Pack

Starter SVG assets for a faceless history explainer style: white background, black ink outlines, and one accent color per faction.

## Style Rules

- Background: pure white `#ffffff`
- Ink: near-black `#111111`
- Secondary ink: muted gray `#6b7280`
- Card stroke: `#e5e7eb`
- Stroke width: 4 px for primary marks, 2.5 px for interior detail at 1600 x 900 sheet scale
- Corners: slightly rounded, not bubbly
- Text: use sparingly in Cavalry, not baked into assets
- Motion style: tokens slide, snap, scale, fade, and rotate like pieces on a strategy board

## Faction Accents

- Byzantium: `#7c3aed`
- Ottomans: `#dc2626`
- Venice: `#0f766e`
- Castile: `#f59e0b`
- France: `#2563eb`
- Ming: `#b91c1c`
- Aztec: `#16a34a`
- Neutral: `#111111`

## Files

- `svg/faction-tokens.svg` - circular faction counters, flags, capital markers
- `svg/map-annotations.svg` - arrows, trade routes, siege rings, timeline pieces
- `svg/historical-objects.svg` - armies, ships, cannons, coins, walls, crowns, grain
- `svg/byzantium-pilot.svg` - production-specific assets for the Byzantium 1453 pilot
- `generated/` - SVG sheets generated by `tools/svg-factory`
- `previews/` - PNG previews generated by `tools/svg-factory`

## SVG Factory

Use the local generator directly when an agent needs to render a one-off SVG:

```bash
svg-factory render icon.json --out icon.svg
npm run svg:render -- icon.json --out icon.svg
```

Use the preset workflow when adding repeatable catalogue assets:

```bash
npm run asset:list
npm run asset:generate
npm run asset:preview
npm run asset:all
```

Sheet definitions live in `tools/svg-factory/sheets/`.
Primitive renderers live in `tools/svg-factory/renderers.mjs`.
The executable style contract lives in `tools/svg-factory/style.mjs`.
Generated SVGs are written to `assets/generated/`, and PNG previews are written to `assets/previews/`.

The generated SVGs are plain editable vectors with stable group IDs, so they can be imported into Cavalry.
Treat generated SVG and PNG files as reproducible outputs; the sheet JSON, renderer modules, validation, tests, and docs are the canonical editable surface.

Preview prefers Inkscape and falls back to macOS `sips`. Renderer differences between Inkscape, `sips`, and Cavalry are a QA signal to inspect the asset, not a reason to hand-edit generated output.

Validation rejects missing sheet metadata, duplicate item IDs, unsupported accents, unknown renderer types, invalid numeric layout values, unsafe labels, and meter values outside 0..1 before writing output.

## Cavalry Workflow

Import the SVG sheet, ungroup the pieces you need, then duplicate them into scenes. Keep the white background empty and let motion carry the explanation.

The light gray catalogue tiles and shadows are for browsing the asset sheets. For final animation scenes, pull the named group out of the tile and animate the asset itself.

Recommended first animation test:

1. Drop in a purple Constantinople wall.
2. Slide red Ottoman army counters around it.
3. Roll in the cannon.
4. Fade out gold coins on the Byzantine side.
5. Draw a red arrow toward the city.
6. End with a verdict card.
